# Security Policy

Security is a top priority for Leaf, especially given its role in managing background system processes and memory. 

## Supported Versions

As an actively developed native macOS utility, Leaf provides security updates for the current major release branch. 
Users are strongly encouraged to keep the app updated to the latest version via the built-in updater.

| Version | Supported          | Notes |
| :-------: | :------------------: | :----- |
| 1.x     | :white_check_mark: | Currently receiving active updates and security patches. |

## Reporting a Vulnerability

If you discover a vulnerability, please report it privately so it can be addressed before public disclosure.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please email: *satwiktungala@gmail.com*

### What to include in your report:
* A detailed description of the vulnerability and its potential impact.
* Step-by-step instructions or a proof-of-concept (PoC) to reproduce the issue.
* The version of Leaf and the version of macOS you are testing on.

### What to expect:

You can expect an initial reply acknowledging your report within 48 to 72 hours.
Following that acknowledgement, the issue will be evaluated. If confirmed, a patch will be developed and distributed to all users via Leaf's built-in Sparkle updater.

Once the vulnerability is resolved and users have had sufficient time to update, you are welcome to publicly disclose your findings. 
I am also happy to credit your discovery in the official release notes.

Thank you for helping keep Leaf safe and optimized!
